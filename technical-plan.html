Why This Works
  • New MzadQatar feature → Government automatically — same codebase, same deploy image
  • Government changes don't affect MzadQatar — routes and controllers isolated behind PARTNER=government
  • Proven pattern — Yemen + Algeria running this way for 2+ years
  • One CI/CD pipeline — build once, deploy to multiple containers with different .env
  • Easier maintenance — security patches, dependency updates apply to all tenants at once
Government is Bigger Than Yemen/Algeria
  • Yemen/Algeria: ~38 partner checks across 15 files (SMS, payments, locale)
  • Government: Same 15 files + 8 new controllers + 8 new DB tables + new routes file + 17 dashboard screens
  • Solution: Isolate all government-specific code in dedicated folders:
    • API: app/Http/Controllers/API/Government/
    • Routes: routes/api_government.php (loaded only when PARTNER=government)
    • Website: resources/js/Pages/Admin/ (dashboard)
    • Migrations: 2026_04_xx_create_government_*

Feature Sync Strategy

New MzadQatar Feature → Government

  1. Feature merged to MzadQatar master
  2. Same image deployed to all containers
  3. If feature is partner-specific, it's behind config('app.partner') check
  4. Government gets it automatically

Government-Only Feature

  1. Code lives in Government/ controller folder
  2. Routes in api_government.php — loaded only when PARTNER=government
  3. No impact on MzadQatar — routes never register
  4. No impact on Yemen/Algeria — same isolation
